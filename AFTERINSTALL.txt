[13-09-22] 1.17.1.0 -> 1.18.0.0
- New built-in variables!
    - INFINITY - A representation of 'infinity'
    - NOT_A_NUMBER - A representation of 'NaN' or 'Not a Number'
- New built-in math functions/variables!
    - Variables
        - MATH_PI - The mathematical constant π = 3.141592... to available precision
        - MATH_TAU - The mathematical constant τ = 6.283185... to available precision
        - MATH_E - The mathematical constant e = 2.71828... to available precision
    - Functions
        - SQUARE_ROOT(value)
              - value: INT or FLOAT
              - Returns square root of 'value'
        - RADIANS_TO_DEGREES(value)
              - value: INT or FLOAT
              - Converts angle 'value' from radians to degrees
        - DEGREES_TO_RADIANS(value)
              - value: INT or FLOAT
              - Converts angle 'value' from degrees to radians
        - COS(value)
              - value: INT or FLOAT
              - Return the cosine of 'value' radians
        - SIN(value)
              - value: INT or FLOAT
              - Return the sine of 'value' radians
        - TAN(value)
              - value: INT or FLOAT
              - Return the tangent of 'value' radians
        - ACOS(value)
              - value: INT or FLOAT
              - Return the arc cosine of 'value', in radians; The result is between 0 and pi
        - ASIN(value)
              - value: INT or FLOAT
              - Return the arc sine of 'value', in radians; The result is between -pi/2 and pi/2
        - ATAN(value)
              - value: INT or FLOAT
              - Return the arc tangent of 'value', in radians; The result is between -pi/2 and pi/2

[14-09-22] 1.18.0.0 -> 1.18.1.0
- New function!
    - SHOW_ERROR(out)
        - message: Any type
        - Raises RUNTIME error with message 'out'
- Bug fix!
    - BUG #1
        - Problem: Single-line TRY statement would not return expression in ERROR sub-statement
        - Status: Fixed
- Other
    - TRY statement tags 'DIVISION-BY-ZERO' and 'MODULO-BY-ZERO' have been merged with the
      'MATH' tag - THERE IS NO BACKWARDS-COMPATABILITY FOR THE PREVIOUS TAGS!
    - New TRY statement tag 'CUSTOM' for errors raised by built-in 'SHOW_ERROR' function
    - Version checks

[19-09-22] 1.18.1.0 -> 1.18.2.0
- String definition change!
    - Strings can now be defined with either ' or "
- Version checker now provides a link to the ezr GitHub page

[19-09-22] 1.18.2.0 -> 1.18.3.0
- Function update!
    - REPLACE(value, arg_a, arg_b)
        - value: STRING|LIST
        - arg_a: STRING|INT
        - arg_b: STRING|Any type
        - STRING REPLACE: Replaces all instances of 'arg_a' with 'arg_b' in
          'value' and returns the STRING
        - LIST REPLACE: Replaces item at index 'arg_a' in list 'value' with
          new item 'arg_b' and returns the new LIST
- Lists can now be difined in multiple lines (check tic-tac-toe demo on GitHub)
- Simpler error messages
- '>>>' -> '>>> ' on the command line

[20-09-22] 1.18.3.0 -> 1.18.4.0
- New functions!
    - GET_TIME(mode)
        - mode: STRING
        - Returns GMTime according to 'mode'
            - mode: 'SECONDS' - Time in seconds since the Epoch (output is a number)
            - mode: 'READABLE' - Time in a readable STRING form ('HOUR:MINUTE:SECOND')
    - GET_DATE()
        - Returns date as a STRING ('DAY:MONTH:YEAR')
- New variable!
    - EZR_VERSION - Version of ezr running the code (as a STRING)
- Version checker updated for new version convention

[21-09-22] 1.18.4.0 -> 1.18.4.1
- Bug fix!
    - BUG #1
        - Problem: If installed ezr version was (for example) 1.18.4.0 and the version in
          the cloud was (for example) 1.17.5.0, the version checker would prompt the user
          to 'update' to 1.17.5.0, even though it's technically a downgrade
        - Status: Fixed

[21-09-22] 1.18.4.1 -> 1.19.0.0
- New comparison expression!
    - IN expression - Used to check if an ITEM contains another ITEM
        - '1' IN '1234567890' -> TRUE (More examples in documentation)
- Better word formatting in version checker

[21-09-22] 1.19.0.0 -> 1.19.0.1
- Bug fix!
    - BUG #1
        - Problem: 'GET(':') IN GET(':')' is invalid as
          the call-expression came before the in-expression in the ezr grammar
        - Status: Fixed
- 'Minor update' -> 'Function update' in version checker

Other news
- Tic-tac-toe demo released on GitHub!
- ezr v1.18.0.0 and greater will also be referred to as ezr m18 and all future feature updates
  will follow a similar convention (m19, m20, m21, ...), though function/patch updates will still be
  referred to as 1.18.1.0, 1.18.2.3, ...
- The official VSCode extension for ezr now supports 1.19.0.0
- New version convention for ezr!
  eg: 1.18.4.1
      Here: 1 -> Major update; 18 -> Feature update; 4 -> Function update; 1 -> Patch
